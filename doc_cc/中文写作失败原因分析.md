# 中文论文生成失败原因分析（为何要求中文仍产出英文）

## 背景
在当前 fork 项目中，即使用户在测试时要求“用中文写论文”，最终生成稿件仍主要为英文。本文基于代码链路排查，分析该现象的主要原因。

## 结论摘要
这不是“中文无法编译”的问题，而是**写作链路缺少语言控制参数**，并且写作/反思提示词与模板上下文整体是英文，导致模型持续回到英文输出。

---

## 1. 写作主流程没有“语言开关”
主入口 `launch_scientist_bfts.py` 的参数中没有 `--language` / `--writeup-language` 之类的选项，调用写作函数时也未传入语言目标。

- 参数定义（无语言参数）：`launch_scientist_bfts.py:43-132`
- 写作调用（未传语言）：`launch_scientist_bfts.py:281-296`

这意味着“中文写作”无法通过标准参数稳定传递到写作阶段。

---

## 2. 写作系统提示词与主提示词默认都是英文语境
两个写作实现（normal / icbinb）的 system prompt 与 writeup prompt 均为英文说明，并未包含“必须中文输出”的硬约束。

### normal 写作链路
- 英文系统提示词模板：`ai_scientist/perform_writeup.py:345-408`
- 英文写作主提示词：`ai_scientist/perform_writeup.py:410-452`
- 英文反思提示词：`ai_scientist/perform_writeup.py:686-705`

### icbinb 写作链路
- 英文系统提示词模板：`ai_scientist/perform_icbinb_writeup.py:533-600`
- 英文写作主提示词：`ai_scientist/perform_icbinb_writeup.py:602-645`
- 英文反思提示词：`ai_scientist/perform_icbinb_writeup.py:1052-1084`、`1127-1147`

模型通常会跟随主系统提示词与大部分上下文语言，因此默认回到英文是预期行为。

---

## 3. 反思循环会持续强化英文输出
写作不是单次生成，而是“生成 -> 反思修订 -> 再生成”的循环。反思提示词仍为英文，会不断把文本风格拉回英文。

在 `icbinb` 写作中还存在一个放大因素：
- 多处调用 `msg_history=msg_history[-1:]`：`perform_icbinb_writeup.py:1091`、`1153`、`1203`

这会缩短历史上下文，若“中文要求”没有被写进系统提示词并在每轮重申，语言偏好更容易丢失。

---

## 4. 模板与默认稿骨架是英文（但这不是唯一根因）
LaTeX 模板内默认章节占位和示例文本均是英文：
- ICML 模板：`ai_scientist/blank_icml_latex/template.tex:60-183`
- ICBINB 模板：`ai_scientist/blank_icbinb_latex/template.tex:53-149`

这会形成额外英文先验，但它主要是“偏置”，不是硬限制。

---

## 5. 不是字体/编译能力导致中文失败
模板已加入 `ctex`，具备中文字符支持：
- `ai_scientist/blank_icml_latex/template.tex:14`
- `ai_scientist/blank_icbinb_latex/template.tex:3`

因此问题核心并非“不能写中文”，而是“上游提示词与参数没有把中文作为稳定目标”。

---

## 根因排序（按置信度）
1. **缺少一等公民语言参数（最高）**：写作阶段无法接收明确语言配置。  
2. **系统提示词/反思提示词全英文（高）**：默认行为自然落到英文。  
3. **反思链路持续英文化（中高）**：多轮修订会覆盖偶发中文片段。  
4. **模板英文占位偏置（中）**：加强英文倾向，但非决定性约束。  

---

## 结论
当前仓库“已具备中文排版能力”，但“未实现中文写作控制链路”。

所以即使用户口头要求中文，系统在自动写作阶段仍会因：
- 无语言参数传递、
- 英文 system prompt + 英文反思循环、
- 英文模板先验，
最终稳定产出英文稿件。
